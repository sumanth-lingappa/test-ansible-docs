netscaler.adc.dnskey module – Configuration for dns key resource.

Note

This module is part of the netscaler.adc collection (version 2.0.0-alpha).

To install it, use: ansible-galaxy collection install netscaler.adc.

To use it in a playbook, specify: netscaler.adc.dnskey.

New in netscaler.adc 2.0.0

Parameters

Parameter

Comments

algorithm

string

Algorithm to generate for zone signing.

Choices:

  • "RSASHA1" ← (default)

  • "RSASHA256"

  • "RSASHA512"

api_path

string

Base NITRO API path.

Define only in case of an ADM service proxy call

bearer_token

string

Authentication bearer token.

Needed when doing an ADM service proxy call.

expires

integer

Time period for which to consider the key valid, after the key is used to sign a zone.

Default: 120

filenameprefix

string

Common prefix for the names of the generated public and private key files and the Delegation Signer (DS) resource record. During key generation, the .key, .private, and .ds suffixes are appended automatically to the file name prefix to produce the names of the public key, the private key, and the DS record, respectively.

instance_id

string

The id of the target NetScaler ADC instance when issuing a Nitro request through a NetScaler ADM proxy.

instance_ip

string

added in netscaler.adc 2.6.0

The target NetScaler ADC instance ip address to which all underlying NITRO API calls will be proxied to.

It is meaningful only when having set mas_proxy_call to true

instance_name

string

The name of the target NetScaler ADC instance when issuing a Nitro request through a NetScaler ADM proxy.

is_cloud

boolean

When performing a Proxy API call with ADM service set this to true

Choices:

  • false ← (default)

  • true

keyname

string

Name of the public-private key pair to publish in the zone.

keysize

integer

Size of the key, in bits.

Default: 512

keytype

string

Type of key to create.

Choices:

  • "KSK"

  • "KeySigningKey"

  • "ZSK" ← (default)

  • "ZoneSigningKey"

mas_proxy_call

boolean

added in netscaler.adc 2.6.0

If true the underlying NITRO API calls made by the module will be proxied through a NetScaler ADM node to the target NetScaler ADC instance.

When true you must also define the following options: nitro_auth_token

When true and adm service is the api proxy the following option must also be defined: bearer_token

When true you must define a target ADC by defining any of the following parameters

instance_ip

instance_id

instance_name

Choices:

  • false ← (default)

  • true

nitro_auth_token

string

added in netscaler.adc 2.6.0

The authentication token provided by a login operation.

nitro_pass

string

The password with which to authenticate to the NetScaler ADC node.

nitro_protocol

string

Which protocol to use when accessing the nitro API objects.

Choices:

  • "http"

  • "https" ← (default)

nitro_timeout

float

Time in seconds until a timeout error is thrown when establishing a new session with NetScaler ADC

Default: 310.0

nitro_user

string

The username with which to authenticate to the NetScaler ADC node.

notificationperiod

integer

Time at which to generate notification of key expiration, specified as number of days, hours, or minutes before expiry. Must be less than the expiry period. The notification is an SNMP trap sent to an SNMP manager. To enable the appliance to send the trap, enable the DNSKEY-EXPIRY SNMP alarm.

Default: 7

nsip

string / required

The ip address of the NetScaler ADC appliance where the nitro API calls will be made.

The port can be specified with the colon (:). E.g. 192.168.1.1:555.

password

string

Passphrase for reading the encrypted public/private DNS keys

privatekey

string

File name of the private key.

publickey

string

File name of the public key.

save_config

boolean

If true the module will save the configuration on the NetScaler ADC node if it makes any changes.

The module will not save the configuration on the NetScaler ADC node if it made no changes.

Choices:

  • false ← (default)

  • true

src

string

URL (protocol, host, path, and file name) from where the DNS key file will be imported. NOTE: The import fails if the object to be imported is on an HTTPS server that requires client certificate authentication for access. This is a mandatory argument

state

string

The state of the resource being configured by the module on the NetScaler ADC node.

enabled and disabled are only valid for resources that can be enabled or disabled.

When present the resource will be created if needed and configured according to the module’s parameters.

When absent the resource will be deleted from the NetScaler ADC node.

When enabled the resource will be enabled on the NetScaler ADC node.

When disabled the resource will be disabled on the NetScaler ADC node.

Choices:

  • "present" ← (default)

  • "absent"

  • "enabled"

  • "disabled"

ttl

integer

Time to Live (TTL), in seconds, for the DNSKEY resource record created in the zone. TTL is the time for which the record must be cached by the DNS proxies. If the TTL is not specified, either the DNS zone’s minimum TTL or the default value of 3600 is used.

Default: 3600

units1

string

Units for the expiry period.

Choices:

  • "MINUTES"

  • "HOURS"

  • "DAYS" ← (default)

units2

string

Units for the notification period.

Choices:

  • "MINUTES"

  • "HOURS"

  • "DAYS" ← (default)

validate_certs

boolean

If false, SSL certificates will not be validated. This should only be used on personally controlled sites using self-signed certificates.

Choices:

  • false ← (default)

  • true

zonename

string

Name of the zone for which to create a key.
